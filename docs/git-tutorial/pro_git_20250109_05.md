至此，我们已经走完了 Git 最基础也最核心的旅程。

让我们回头看看这几章里我们都装备了什么技能：
*   你可以创建和克隆仓库；
*   你可以自如地进行每日的修改、暂存和提交；
*   你学会了用 `git log` 像侦探一样审视过去的历史；
*   你懂得了如何用 `git remote` 和世界各地的开发者协作；
*   你知道了在搞砸的时候如何用“后悔药”来修正；
*   甚至，你还学会了如何给里程碑打标签，以及如何通过别名让 Git 变得更顺手。

说实话，掌握了这些，你已经超越了 90% 只会 `commit` 和 `push` 的 Git 用户。在很多简单的项目中，这些知识已经完全够用了。

**但是，这只是 Git 的冰山一角。**

Git 真正让它与其他版本控制系统（如 SVN）拉开维度的，是它的**分支模型**。
你可能听说过 Git 的分支既轻量又快，或者听说过 Git Flow 工作流。接下来的章节，我们将进入 Git 最精彩的部分——**分支 (Branching)**。

那是平行宇宙交汇的地方，是现代软件工程多人协作的基石。准备好了吗？我们下一章见。
