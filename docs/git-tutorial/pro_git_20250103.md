本节介绍 Git 的核心概念。理解这些概念是掌握 Git 的关键，因为 Git 在对待数据的方式上，与 Subversion (SVN) 等其他版本控制系统有着本质的区别。

<!-- truncate -->

## 核心区别：快照 (Snapshot) 与 差异 (Delta)

这是 Git 与其他 VCS 最主要的区别：

*   **其他系统 (如 SVN)**：将数据看作是一组文件的变化。它们记录的是每个文件在不同版本之间的差异（Delta-based）。
*   **Git**：将数据看作是小型文件系统的一组**快照**。
    每次提交更新时，Git 都会对当前的全部文件制作一个快照并保存这个快照的索引。为了效率，如果文件没有修改，Git 不会重新存储该文件，而是只保留一个链接指向之前存储的文件。

## 近乎所有的操作都是本地执行

在 Git 中的绝大多数操作都只需要访问本地文件和资源，一般不需要来自网络上其它计算机的信息。

*   **速度优势**：因为不需要网络延迟，像浏览项目历史、比较版本差异等操作几乎是瞬间完成的。
*   **离线工作**：在没有网络的情况下（如飞机上、火车上），你依然可以进行提交（Commit）、创建分支和查看日志，待有网络时再推送到远程仓库。

## Git 保证完整性

Git 中所有的数据在存储前都会计算校验和（Checksum），然后以校验和来引用。这意味着不可能在 Git 不知情的情况下更改文件内容或目录结构。

Git 使用 **SHA-1** 算法计算校验和，其结果是一个 40 位的十六进制字符串。Git 数据库中也是通过通过文件内容的哈希值来索引，而不是文件名。

## Git 的三种状态

这是 Git 中最重要的概念之一。Git 有三种主要的状态，你的文件可能处于其中之一：

1.  **已提交 (Committed)**：数据已经安全地保存在本地数据库中。
2.  **已修改 (Modified)**：修改了文件，但还没保存到数据库中。
3.  **已暂存 (Staged)**：对一个已修改文件的当前版本做了标记，使之包含在下次提交的快照中。

由此引入了 Git 项目的三个工作区域：
*   **Git 仓库 (Repository)**：用来保存项目的元数据和对象数据库的地方。
*   **工作目录 (Working Directory)**：对项目的某个版本独立提取出来的内容。
*   **暂存区 (Staging Area)**：保存了下次将提交的文件列表信息（也称为 Index）。
