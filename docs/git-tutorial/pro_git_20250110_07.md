这一章我们深入探讨了 Git 的杀手级特性——分支。

如果说前面的基础命令让你成为了一个会用工具的操作员，那么对“分支”的理解，则标志着你开始像更高级的工程师一样思考。

**我们学到了：**
1.  **轻量级原理**：分支只是一个指向 commit 的 40 字节指针，所以它创建和切换都极快。
2.  **基本操作**：新建、切换、合并、删除。这四个动作将贯穿你的职业生涯。
3.  **解决冲突**：不再恐惧 `CONFLICT`，而是把它看作是代码的一场对话。
4.  **远程协作**：理解了 `origin/master` 和本地 `master` 的区别，知道了 `fetch` 和 `pull` 的深层含义。
5.  **Rebase vs Merge**：那是关于“真实历史”与“整洁历史”的哲学选择，以及那条不可逾越的“黄金法则”。

分支模型改变了我们开发软件的方式。
它让我们敢于尝试，敢于犯错，因为我们知道主线永远是安全的。
它让多人协作变得有序，每个人都可以在自己的平行宇宙里构建大厦，最后无缝地拼接到一起。
