程序员的三大美德是什么？懒惰、急躁、傲慢。
这里的“懒惰”是指：如果有办法能让我少敲几次键盘，我绝不多敲一次。

Git 的命令有时候确实有点长。比如 `git checkout`，`git commit`，`git status`。每天敲一百遍 `git checkout`，手指也会抗议。

Git 别名 (Alias) 就是为了解决这个问题而生的。它允许你为较长的命令设置简写。这不仅仅是少敲几个字，更是为了让工具顺应你的思维速度，而不是阻碍它。

<!--truncate-->

### 基础设置

你可以通过 `git config` 来轻松设置。

```bash
$ git config --global alias.co checkout
$ git config --global alias.br branch
$ git config --global alias.ci commit
$ git config --global alias.st status
```

设置完之后：
*   想切换分支？ `git co main`
*   想看状态？ `git st`

你会发现，这些微小的改变会带来极大的流畅感。这就好比不管是玩游戏还是用 IDE，高手总是会把自己常用的操作绑定到顺手的快捷键上。

### 进阶玩法：取消暂存

还记得我们在 2.4 节提到的 `git restore --staged` 或者老式的 `git reset HEAD` 吗？这两个命令都太难记了。
我们可以给它起个别名，就叫 `unstage`：

```bash
$ git config --global alias.unstage 'reset HEAD --'
```

现在，想把文件移出暂存区，只需要：
```bash
$ git unstage file.txt
```
瞬间清爽了，对吧？

### 骨灰级玩法：可视化日志

`git log` 的默认输出虽然详细，但如果不加参数，很难看清分支的走向。
你可以配置一个超级别名，我管它叫 `git last` 或者 `git lg`：

```bash
$ git config --global alias.lg "log --color --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"
```

这行命令看起来像是乱码，但请把它复制到你的终端里运行一次。
然后输入 `git lg`。

效果会让你惊艳的。它会画出一棵彩色的树，显示提交哈希、分支指针、提交信息、相对时间（如“2天前”）和作者。这是我每天使用频率最高的别名，没有之一。

### 总结

Git 别名不仅仅是缩写，它是一种**定制化**。
不要小看这些配置。工欲善其事，必先利其器。当你甚至感觉不到工具的存在，指令如流水般从指尖流出时，那才是编程的至高境界。
