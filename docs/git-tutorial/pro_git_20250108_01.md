至此，我们完成了《Pro Git》第一章的旅程。
在这个充满了理论概念、历史背景和环境配置的章节里，我们几乎没有写下一行实际的代码。很多急于求成的初学者可能会觉得这一章枯燥乏味，甚至想要跳过。

但在我看来，**这一章恰恰是区分“Git 操作员”和“Git 工程师”的分水岭。**

<!--truncate-->

## 为什么基础如此重要？

Git 的学习曲线之所以被公认为陡峭，是因为它试图用一种**全新的思维模型**来替代你已有的认知（比如复制粘贴备份，或者 SVN 的差异模型）。如果你不先花时间去重塑这个模型，那么后面遇到的每一个命令（Branch, Rebase, Reset）都会让你感到困惑和恐惧。

## 核心概念回顾

让我们抛开细节，再次审视那些能够支撑你走得更远的基石：

1.  **快照流 (Snapshot Stream)**
    这是 Git 的灵魂。Git 不记录“怎么变”，只记录“变成了什么”。这种空间换时间的策略，赋予了 Git 极高的性能和数据完整性。

2.  **三种状态 (Three States)**
    **已提交**是历史，**已修改**是现状，而**已暂存**则是你精心编排的未来。理解了这三者如何在工作目录、暂存区和仓库之间流转，你就理解了 Git 的每一次心跳。

3.  **身份与责任**
    `git config` 不仅仅是设置，更是确立你在协作网络中的身份凭证。

4.  **去中心化的自信**
    由于本地拥有完整的仓库镜像，你可以无所畏惧地在本地进行任何实验。因为你知道，只要不推送到远程，任何错误都是可以被“修正”或“重置”的。
