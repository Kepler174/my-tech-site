至此，我们要完成了 《Pro Git》第一章的学习。本章主要涵盖了 Git 的基础理论和环境准备，为后续的实际操作打下了基础。

<!--truncate-->

## 核心知识点回顾

1.  **快照流 (Snapshot Stream)**：
    Git 与其他版本控制系统的根本区别在于，Git 记录的是项目文件的全量快照，而非差异变化。
2.  **三种状态 (Three States)**：
    理解 **已提交 (Committed)**、**已修改 (Modified)** 和 **已暂存 (Staged)** 的区别，以及工作目录、暂存区和 Git 仓库之间的流转关系，是使用 Git 的核心。
3.  **环境配置**：
    正确配置 `user.name` 和 `user.email` 是使用 Git 的前提。
4.  **获取帮助**：
    学会使用 `git help <command>` 或 `git <command> -h` 来查阅文档。

在接下来的章节中，我们将进入实战环节，学习如何创建仓库、跟踪文件修改以及提交更新等日常操作。
