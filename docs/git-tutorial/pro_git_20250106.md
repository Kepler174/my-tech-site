安装完 Git 并不意味着我们可以立刻开始工作。这就好比买了一辆新车，上路前你得先调整好座椅、后视镜，并输入导航回家的地址。
Git 需要知道“你是谁”，以及你习惯“怎么工作”。

这些配置只需要做一次，但它们将伴随你的整个开发生涯。

<!--truncate-->

## 数字化你的身份

在分布式协作网络中，信任的基础是身份。Git 的每一次提交（Commit）都会永久性地烙印上作者的名字和邮箱。

请务必慎重地在终端输入以下命令：

```bash
git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"
```

这里的 `--global` 参数意味着，除非你在具体的项目中另行指定，否则这台机器上所有的 Git 操作都将代表你本人。

**经验分享**：
经常有新人问我：“这里的邮箱需要和 GitHub 账号的邮箱一致吗？”
答案是：最好一致。GitHub 等代码托管平台正是通过这个邮箱地址，将代码提交记录关联到你的账户头像上的。如果你这里乱填，虽然提交不会报错，但在 GitHub 的贡献图（绿格子）上，这些代码可能就不会算在你头上了。

## 选择趁手的兵器

当 Git 需要你输入这类长篇大论的信息时（比如撰写提交说明），它会自动调用系统默认的文本编辑器。
在 Linux 上这通常是 Vi 或 Vim。如果你还没做好准备面对 Vim 陡峭的学习曲线，可能会陷入“无法保存退出”的窘境。

我强烈建议你将其配置为你熟悉的现代编辑器，比如 VS Code：

```bash
git config --global core.editor "code --wait"
```
这样，Git 就会弹出一个 VS Code 窗口让你输入信息，并在你关闭文件后继续执行。

## 查看你的设置

如果你想检查刚才的配置是否生效，可以使用：

```bash
git config --list
```
这会列出所有 Git 能找到的配置。有时候你会发现同一个配置出现了两次，那是因 Git 会依次读取系统级、用户级和仓库级的配置文件。以最后一个出现的为准。

配置 Git 的过程，其实就是根据自己的习惯打磨工具的过程。一把趁手的兵器，能让你在代码世界中挥洒自如。
