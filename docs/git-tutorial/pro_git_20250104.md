在学习 Git 时，我们有多种工具可供选择，包括传统的命令行工具（CLI）和各种现代化的图形用户界面（GUI）。

<!--truncate-->

## 为什么要使用命令行？

虽然图形界面工具（如 SourceTree, GitHub Desktop, VS Code 内置 Git）在日常操作中非常直观便捷，但命令行工具依然是学习 Git 的最佳途径，原因如下：

1.  **功能完整性**：
    命令行是唯一支持 Git **所有**命令的地方。许多高级功能和复杂操作（如复杂的 Rebase、二分查找等）在 GUI 中并没有完全实现或难以找到。

2.  **通用性**：
    掌握了命令行，你可以在任何操作系统、任何服务器环境中使用 Git，而无需依赖特定的图形软件。

3.  **理解原理**：
    通过输入命令，能更清晰地理解 Git 的工作流程和操作逻辑。

因此，本教程将主要通过终端命令行来讲解 Git 的使用。当你熟悉了这些命令后，自然也能更游刃有余地使用各种图形化工具。
