很多新人入坑 Git 时都会问：“现在 SourceTree、GitHub Desktop 甚至 VS Code 自带的 Git 工具这么好用，为啥还要苦哈哈地背命令行？”

这也是我当年的疑问。但用了几年后，我变成了“命令行吹”。

<!--truncate-->

## 图形界面是“残血版”

GUI 工具为了界面简洁，通常只把最常用的功能（commit, push, pull, merge）做成了按钮。
但 Git 有很多高级功能（比如交互式 rebase、二分查找 bug、精细的分支管理），在图形界面里要么找不到，要么藏得很深。

**命令行才是 Git 的完全体**。只有在终端里，你才能发挥 Git 100% 的威力。

## 通用性才是王道

*   你会了 SourceTree，换家公司不让你装盗版软件咋办？
*   你会了 VS Code 的 Git 插件，换台 Linux 服务器或者只有终端的测试环境咋办？
*   **但 `git commit` 在哪都是 `git commit`。**

学会了命令行，你就掌握了底层逻辑，再去看任何图形工具，无非就是“这个按钮对应那个命令”而已，上手极快。反之则不然。

## 给新手的建议

刚开始接触，确实记不住那么多参数。我的建议是：
**强迫自己用一段时间终端**。哪怕是最简单的 `git status` 和 `git add`。
遇到复杂的 ihr rebase 或者看非常复杂的 diff，我也用 GUI，这不丢人。**工具是为了效率服务的，谁快用谁，但底层原理得懂。**

接下来的笔记，我都会用命令行来演示，咱们一起习惯在黑底白字中操作的感觉。
