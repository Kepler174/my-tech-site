这节讲 Git 的历史，其实挺有意思的。Git 并不是哪个公司为了商业目的精心设计的，而是 Linus Torvalds（Linux 之父）**特别是生气**的时候搞出来的。

<!-- truncate -->

## 被逼出来的神器

直到 2002 年，Linux 内核这么大个项目，居然都没用正经的版本控制工具。Linus 觉得当时的工具（像 CVS）都太难用了，根本配不上 Linux。

后来他们试用了一个叫 **BitKeeper** 的商业软件，虽然好用，但是要钱。BitMover 公司看在 Linux 面子上，让他们免费用了三年。

结果 2005 年，社区里有人试图破解 BitKeeper 的协议，把人家惹毛了：“免费给你们用还搞破解？收回授权！”
这下 Linux 源码没地儿放了。

Linus 也是个暴脾气，他看了一圈市面上的工具，觉得没一个能打的。于是他决定自己造轮子。他的要求很“变态”：
1.  **必须要快**（Linux 内核太大了，慢一点会疯）。
2.  **必须是分布式的**（不仅是为了安全，也是为了不被卡脖子）。
3.  **要支持非线性开发**（成千上万个分支并行）。

## 两周封神

最离谱的是，Linus 大概只用了**两周**就写出了 Git 的原型。两周啊！很多人两周连个 PPT 都写不完。
紧接着，这个工具就被用来管理 Linux 源码了。

## 我的看法

Git 的诞生故事其实告诉我们两件事：
1.  **核心工具必须掌握在自己手里**。如果当初 Linux 继续依赖 BitKeeper，后果不堪设想。
2.  **大神就是大神**。觉得工具不顺手就自己写一个更好的，这种解决问题的思路和执行力，才是我们该学的。虽然我们写不出 Git，但写个自动脚本提升下效率还是可以的嘛。
