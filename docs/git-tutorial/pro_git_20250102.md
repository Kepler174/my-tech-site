Git 的诞生与 Linux 内核的开发密切相关。了解 Git 的历史，有助于理解其设计哲学：速度、简单的设计、对非线性开发模式的强力支持、完全分布式以及有能力高效管理超大规模项目。

<!-- truncate -->

## 诞生背景

Linux 内核开源项目在早期的维护过程中，并没有使用正式的版本控制工具。直到 2002 年，Linux 项目组开始使用一个名为 **BitKeeper** 的分布式版本控制系统。

到了 2005 年，由于开发社区与 BitKeeper 的商业公司之间发生了关系破裂，Linux 社区失去了免费使用 BitKeeper 的权利。这迫使 Linux 社区（特别是 Linux 之父 Linus Torvalds）必须要开发一套属于自己的版本控制工具。

## 设计目标

基于对 BitKeeper 的使用经验，以及对已有系统的不足之处的反思，Linus 对新工具提出了明确的目标：

1.  **速度**：必须要极快，以适应内核开发庞大的文件量。
2.  **简单的设计**：架构清晰简洁。
3.  **非线性开发支持**：必须有力支持数千个并行的分支开发。
4.  **完全分布式**：每个开发者都有完整的仓库副本。
5.  **高效管理**：能够像 Linux 内核一样高效地管理大规模项目。

## 快速发展

Git 的原型开发非常迅速。自 2005 年诞生以来，Git 已经逐渐完善并成为了最流行的版本控制系统。它保留了最初设计时的灵活性和速度，同时在功能和易用性上不断进化，成为了现代软件开发的标准工具。
