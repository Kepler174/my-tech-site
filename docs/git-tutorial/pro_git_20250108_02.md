本章开始介绍 Git 的基础操作。通常有两种方式来获取一个 Git 仓库：一种是将本地现有的目录转换为 Git 仓库，另一种是从服务器克隆现有的 Git 仓库。

<!--truncate-->

## 在现有目录中初始化仓库 (git init)

如果你有一个尚未进行版本控制的项目目录，想要用 Git 来管理它，需要执行以下步骤：

1.  进入项目目录：
    ```bash
    cd my_project
    ```
2.  执行初始化命令：
    ```bash
    git init
    ```

该命令会在当前目录下创建一个名为 `.git` 的子目录，其中包含了 Git 仓库的所有必需文件（骨架）。此时，项目中的文件还没有被跟踪。你需要使用 `git add` 和 `git commit` 来进行初次提交。

## 克隆现有的仓库 (git clone)

如果你想获取一份已经存在的 Git 仓库（例如 GitHub 上的开源项目）的拷贝，需要使用 `git clone` 命令。

```bash
git clone https://github.com/libgit2/libgit2
```

执行该命令后，Git 会做以下几件事：
1.  在当前目录下创建一个名为 `libgit2` 的文件夹。
2.  初始化一个 `.git` 目录。
3.  **下载该仓库的所有数据**（包括所有历史版本）。
4.  检出最新版本的代码到工作目录。

如果你想在这个命令中自定义本地仓库的名字，可以指定第二个参数：

```bash
git clone https://github.com/libgit2/libgit2 mylibgit
```

Git 支持多种传输协议，最常见的是 `https://` 协议，此外还支持 `git://` 或 `ssh://` 等协议。
