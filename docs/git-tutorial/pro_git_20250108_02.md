这章是实战篇的开始。
获取 Git 仓库说白了就两招：要么是你自己新开个项目（`init`），要么是你去把别人的项目搬回家（`clone`）。

<!--truncate-->

## 方式一：白手起家 (git init)

如果你有个项目想开始用 Git 管理：

1.  进入那个文件夹： `cd my_project`
2.  敲下神圣的： `git init`

你会发现多了一个 `.git` 文件夹（看不到是因为隐藏了）。
**重点**：这时候你的代码还没被 Git 管起来呢！`init` 只是修好了仓库这个“房子”，你还得把家具（文件）搬进去（add & commit）。

## 方式二：全盘拷贝 (git clone)

这也就是我们常说的“拉代码”。但 Git 的 Clone 比 SVN 的 Checkout 猛多了。
SVN Checkout 只是拿到了最新的那份代码。
Git Clone 是把人家**祖宗十八代的所有历史记录**全都搬到你电脑上了。

```bash
git clone https://github.com/libgit2/libgit2
```

**我的经验**：
大部分时候我们都是用 `git clone`。
有个小技巧，如果你想克隆下来后改个文件夹名字（比如不想叫 libgit2，想叫 my-lib），直接加在后面就行：

```bash
git clone https://github.com/libgit2/libgit2 my-lib
```

这就省得克隆完再去重命名了，多优雅。
至于协议，HTTPS 最简单无脑，SSH 适合配置好密钥的老手（不用每次输密码）。新手建议先用 HTTPS 跑通再说。
